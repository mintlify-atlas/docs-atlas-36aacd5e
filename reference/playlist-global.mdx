---

`PlaylistGlobal` is a `static` class that holds a single in-memory playlist — a `List<Cancion>` together with an integer index tracking the currently playing song. Because it is `static`, every form in the application reads from and writes to the exact same state without needing to pass object references between forms or use dependency injection. `Form1` (the file browser) adds songs via `AgregarCancion`, while `FormMP3` (the media player) reads the list with `ObtenerPlaylist` and advances playback through `EstablecerCancionActual`.

---

## Static Methods

### AgregarCancion()

```csharp
public static void AgregarCancion(string rutaArchivo)
```

Appends a new `Cancion` to the playlist if the file exists on disk and is not already present. Duplicate detection is case-insensitive (`OrdinalIgnoreCase` path comparison). If `rutaArchivo` is null, empty, or points to a non-existent file, the method returns silently without throwing.

<ParamField path="rutaArchivo" type="string" required>
  Absolute path to the MP3 file. Used as both the deduplication key and the `Cancion.Ruta` value. `Cancion.Nombre` is derived from `Path.GetFileNameWithoutExtension(rutaArchivo)`.
</ParamField>

---

### ObtenerPlaylist()

```csharp
public static List<Cancion> ObtenerPlaylist()
```

Returns the internal `List<Cancion>` directly. The list is shared — modifications to the returned reference affect the global state.

---

### ObtenerCancionActual()

```csharp
public static Cancion? ObtenerCancionActual()
```

Returns the `Cancion` at the current index, or `null` if the playlist is empty or no song has been selected yet (index = -1).

---

### EstablecerCancionActual()

```csharp
public static void EstablecerCancionActual(string rutaArchivo)
```

Finds the song whose `Ruta` matches `rutaArchivo` (case-insensitive) and updates the internal current index to that position. If the song is not found, the index is left unchanged.

<ParamField path="rutaArchivo" type="string" required>
  The full file path of the song to make current. Must already be in the playlist.
</ParamField>

---

### ObtenerIndiceActual() / EstablecerIndiceActual()

```csharp
public static int ObtenerIndiceActual()
public static void EstablecerIndiceActual(int indice)
```

Direct index accessors. `FormMP3` uses these when implementing previous/next track navigation, incrementing or decrementing the index and then calling `ObtenerCancionPorIndice` to retrieve the target song.

<ParamField path="indice" type="int">
  Zero-based index into the playlist. No bounds validation is applied — callers should check `ObtenerCantidad()` before setting.
</ParamField>

---

### ObtenerCancionPorIndice()

```csharp
public static Cancion? ObtenerCancionPorIndice(int indice)
```

Returns the `Cancion` at the given index, or `null` if the index is out of range.

---

### ObtenerCantidad()

```csharp
public static int ObtenerCantidad()
```

Returns the total number of songs currently in the playlist.

---

### Limpiar()

```csharp
public static void Limpiar()
```

Removes all songs from the playlist and resets the current index to `-1`. Typically called when the user starts a new session or clears the queue.

---

## Cancion Model

```csharp
public class Cancion
{
    public string Nombre { get; set; }
    public string Ruta   { get; set; }

    public override string ToString() => Nombre;
}
```

<ResponseField name="Nombre" type="string">
  Display name shown in the playlist UI. Automatically set to `Path.GetFileNameWithoutExtension(rutaArchivo)` by `AgregarCancion()`. `ToString()` returns this value so `Cancion` instances can be bound directly to `ListBox` or `ComboBox` controls.
</ResponseField>

<ResponseField name="Ruta" type="string">
  The full absolute file path. Serves as the primary key for deduplication and for locating the file during playback.
</ResponseField>

---

## Why Static?

WinForms applications do not have a built-in service container or navigation stack, so sharing state between two simultaneously open forms requires either passing an object reference at construction time or using a static holder. `PlaylistGlobal` uses the static approach: both `Form1` and `FormMP3` are independent top-level windows that can be opened in any order, making constructor injection fragile. The static class ensures that both forms always see exactly the same playlist regardless of instantiation order.

---

## Usage Example

```csharp
// In Form1 — user double-clicks an MP3 file in the file browser
private void listBox_DoubleClick(object sender, EventArgs e)
{
    string selectedPath = GetSelectedFilePath();

    // Add to shared playlist (no-op if already present)
    PlaylistGlobal.AgregarCancion(selectedPath);

    // Mark as the song to play immediately
    PlaylistGlobal.EstablecerCancionActual(selectedPath);

    // Open the player form (it will read state from PlaylistGlobal)
    var player = new FormMP3();
    player.Show();
}

// In FormMP3 — populate the playlist list box on load
private void FormMP3_Load(object sender, EventArgs e)
{
    var playlist = PlaylistGlobal.ObtenerPlaylist();
    listBoxCanciones.DataSource = playlist; // Cancion.ToString() provides display text

    Cancion? current = PlaylistGlobal.ObtenerCancionActual();
    if (current is not null)
        PlaySong(current.Ruta);
}

// In FormMP3 — next track button
private void btnSiguiente_Click(object sender, EventArgs e)
{
    int next = PlaylistGlobal.ObtenerIndiceActual() + 1;
    if (next < PlaylistGlobal.ObtenerCantidad())
    {
        PlaylistGlobal.EstablecerIndiceActual(next);
        Cancion? song = PlaylistGlobal.ObtenerCancionPorIndice(next);
        if (song is not null) PlaySong(song.Ruta);
    }
}
```
